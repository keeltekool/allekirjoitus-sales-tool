Feature Introduction

Enterprise Signing Capabilities

Prepared for KPMG Finland

Allekirjoitus.fi is a cloud-based electronic signing platform by SK ID Solutions, purpose-built for enterprise organizations in Finland and the Nordics. This document outlines the platform's core capabilities, workflow orchestration options, and enterprise configuration features relevant to your organization.

Platform Foundation

Core Signing Service

Qualified & Advanced Electronic Signatures

eIDAS-compliant digital signing across the EU. Every signed document receives a qualified timestamp and integrity seal via an AATL-certified certificate, ensuring long-term legal validity and tamper-proof evidence. Recipients see a trusted validation indicator in standard Adobe Acrobat — no plugins needed. Documents use PAdES-LTV format, remaining independently verifiable offline for decades.

Authentication Methods

Your signers use their own familiar bank credentials or national eID — no account creation, no software, no registration. Included: Finnish Trust Network (all major banks + Mobile Certificate), Swedish BankID, Freja eID. Available as add-on: Norwegian BankID, Vipps, Danish MitID, Baltic Smart-ID, Mobile-ID, Estonian e-Residency. QES via national ID cards for the highest legal tier.

Portal & REST API

Full-featured web portal for document management — create, send, track, download. Complete REST API for end-to-end automation: programmatic request creation, webhook-driven status updates, embedded signing within your own applications, and automated archiving. Some Enterprise customers operate entirely API-only.

Audit Trail & Compliance

Every signed document is self-proving. Complete evidence record per signatory — identity, authentication method, timestamps, IP address — embedded inside the PDF. Qualified timestamps, PAdES-LTV long-term verification. GDPR-ready: you are Data Controller, SK is Data Processor. SSN can be cryptographically hashed.

Process Control

Workflows & Orchestration

Signing Order & Approval Hierarchies

Simultaneous signing for straightforward agreements. Sequential signing for approval chains (employee → manager → director). Mixed mode for complex hierarchies. Multi-level signatory groups with cascade rejection — if a lower level rejects, upper levels are never notified.

Signature Request Templates

Predefined workflow blueprints that standardize business processes. Each template pre-configures authentication strength, delivery methods, signing order, and compliance rules. One template for employment contracts, another for NDAs, a third for loan agreements — all under one tenant. Compliance-critical rules can be locked so portal users cannot override them.

Review Request (Reflection Period)

Regulatory compliance workflow enforcing a mandatory waiting period before signing. The signer reads the document immediately, but the Sign button stays disabled until the period expires (overnight, 7 days, 14 days). Designed for financial sector — actively deployed for consumer loan compliance.

Advanced Workflow Features

Signatory forwarding for delegation without restarting. Watchers for compliance oversight without signing rights. Signer document upload (ID copies, diplomas) during the ceremony. Post-sign redirect back to your CRM. Custom metadata for downstream routing. Bundle signing — authenticate once, sign up to 50 documents.

Your Organization

Enterprise Configuration

Custom Branding & Domain

Your logo, colors, and fonts on both the sender portal and signer-facing pages. Branded PAdES cover page. Custom domain (signing.yourcompany.fi) with SSL. Per-department branding variations. Signers may not realize a third-party platform is involved.

SSO / Active Directory Integration

OIDC with Azure AD / Entra ID. Instant sign-in if already authenticated with Microsoft. Auto-provisioning on first login. Role changes sync automatically. AD group removal revokes access immediately — no manual action in the signing portal.

Organizational Structure & Roles

Tenant → Department → Group → User hierarchy mapped from Active Directory. Role-based access: Organization Admin (full visibility), Department Admin (department scope), Department User (own requests). Unlimited users included.

How Insurance Companies Use the Platform

Sequential signing with review/reflection periods for regulatory compliance on financial products. Dynamic metadata tracking signatory roles (Applicant, Guarantor) for automatic CRM routing. Open forms for public-facing insurance applications with mandatory strong authentication — the applicant fills a cascading web form and is redirected straight to signing. Automated archiving to secure vaults upon completion.
