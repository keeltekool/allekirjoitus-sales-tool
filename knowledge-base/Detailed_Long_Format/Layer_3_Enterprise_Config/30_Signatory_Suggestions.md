# Signatory Suggestions

When your team creates signing requests for the same counterparties repeatedly, the platform remembers previously used contacts and auto-completes their details - eliminating repetitive data entry and preventing typos that cause bounced emails or authentication failures.

- **How auto-complete works in practice** - when a sender reaches the "Add Signatory" step while creating a new request, the portal offers suggestions based on previously entered contacts. Instead of manually typing a person's full name, email, phone number, and identity code from scratch, the sender can select from a list of previous signatories and instantly populate all required fields. This is particularly valuable for senders who regularly route documents to the same executives, board members, or business partners

- **SSN-based lookup** - for organizations that enforce identity code collection, the platform offers an advanced lookup mode. The sender can type just the first digits of a Personal Identity Code into the SSN field, and the portal automatically suggests previous signatories whose identity codes match that prefix. This allows precise identification of returning contacts even when the sender does not remember the person's exact email or phone number

- **What data is retained** - the suggestion system retains previously entered signatory details including names, email addresses, phone numbers, and SSNs. This data is stored within the platform and linked to the tenant's configuration, ensuring suggestions are available across sessions

- **Workflow acceleration** - for repeat-heavy processes like quarterly board approvals, recurring supplier agreements, or regular HR workflows, signatory suggestions transform request creation from a multi-field manual entry task into a quick selection. Combined with Signature Request Templates that pre-configure all workflow rules, this means your senders can dispatch a fully configured, compliance-correct signing request in seconds
