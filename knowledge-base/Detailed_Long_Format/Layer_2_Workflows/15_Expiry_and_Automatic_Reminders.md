# Expiry & Automatic Reminders

Every signature request carries a hard expiration deadline. Combined with configurable automatic reminders and manual nudge capabilities, this ensures your documents are signed on time or cleanly closed when they are not.

- **How expiration works** - the sender sets a fixed expiry date when creating the request (e.g., 7, 14, 30, or 90 days). When the deadline passes before all required signers complete, the request status automatically changes to "Expired" and the document is permanently closed. Expired requests cannot be extended or reopened - if the document still needs to be signed, the sender must create a new request with a fresh deadline

- **Default reminder schedule** - the platform sends automatic reminder emails on a standard schedule: the first reminder 2 days after the request is dispatched, and a final warning 1 day before the expiry date. The reminder contains the same secure signing link as the original invitation, so the signer has everything they need to complete without searching for the original email

- **Enterprise customization** - Enterprise customers can fully customize automatic reminder schedules with configurable intervals, timing reminders either counting forward from dispatch (e.g., every 3 days) or backward from the expiry deadline. Reminders can be sent via email (standard) or SMS (additional per-message fee)

- **Manual reminders** - senders can trigger on-demand ad-hoc reminders to individual signers directly from the portal dashboard. This is also used after correcting a bounced email address to immediately resend the invitation. Manual reminder capability is an Enterprise feature and is not available on the Basic/Portal tier

- **Sequential signing interaction** - in sequential workflows, automatic reminders are dynamically targeted only at the currently active signer whose turn it is. Upper-tier signers waiting for their turn do not receive premature reminder emails
