# File Sharing

When you need to securely distribute documents to recipients who should read but not sign, File Sharing delivers them through the platform's encrypted infrastructure with OTP-protected access - a controlled alternative to email attachments for sensitive materials.

- **How secure sharing works** - instead of adding a recipient as a signatory, you use the File Sharing feature to transmit a document for viewing and downloading only. The document is hosted within the platform's secure environment, encrypted at rest and in transit. The recipient receives an email with a secure link rather than the file itself

- **OTP-protected access** - when the recipient clicks the download link, they must verify their identity by entering a one-time password delivered via SMS to their registered mobile phone. Only after passing this two-factor check (email link plus SMS OTP) can they access and download the document. This ensures the file reaches only the intended recipient, even if the email is forwarded or the inbox is compromised

- **How it differs from email attachments** - standard email attachments can be intercepted, forwarded to unauthorized parties, or accessed if an inbox is breached. File Sharing keeps the document within the platform's secure boundary until the exact intended recipient proves their identity via their mobile device, maintaining a significantly higher level of control over sensitive content

- **Use cases** - distributing finalized, executed contracts to external legal counsel, auditors, or third-party stakeholders who need to review the agreement but are not signatories. Sending confidential appendices, financial reports, or background data to clients with SMS OTP protection before they can access the content. Delivering signed documents to compliance teams or regulatory bodies who require a secure receipt mechanism

- **Enterprise add-on** - File Sharing is billed as a transactional add-on per recipient
