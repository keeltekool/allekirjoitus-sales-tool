# Allekirjoitus.fi Service Terms Summary

# PART 1: Key Service Terms

## 1.1 Service Provider
**SK ID Solutions AS** provides Allekirjoitus.fi as a cloud-based electronic signing service. These terms apply alongside the General Terms of Subscriber Agreement.

## 1.2 Billing Unit — Signature Request
**One Signature Request (SR) is charged per signatory** when a document is dispatched for signing — regardless of whether the signatory completes or declines. A document sent to four signatories consumes four SRs.

## 1.3 Enterprise Billing Model
**Monthly Platform Base Fee + per-transaction fees + SMS charges.** No fixed packages — transparent usage-based billing. All prices per the current Allekirjoitus.fi Price List (VAT 0%).

## 1.4 Document Requirements
**PDF-A format required** for long-term archival compatibility. Documents must not be password-protected. Maximum file size: 100 MB per document.

## 1.5 Signed Document Format
**PAdES format with qualified timestamp and AATL-compatible seal.** Every signed document includes an XML audit trail. Independently verifiable in Adobe Reader — no external service needed.

## 1.6 Document Retention
**90-day default retention** from request completion. The Subscriber must transfer all signed documents to their own storage before expiry. SK does not provide long-term archival.

## 1.7 Data Processing
**Subscriber is data controller; SK is data processor.** Personal data processed: name, email, phone, identity code (HETU), IP address, authentication data. HETU is stored in XML metadata only — not visible on document pages.

## 1.8 Enterprise Access & Authentication
**Unlimited users. SSO via Azure AD / Active Directory.** Auto-provisioning on first SSO login. Advanced user rights with department structures from AD. AES default; optional SES (SMS OTP, email OTP, drawn signature).

## 1.9 Support & Service Level
**Technical support Mon–Fri, 09:00–17:00 EET** via support@allekirjoitus.fi. Service provided on best-effort basis due to dependency on third-party authentication providers. No SLA penalties apply.

## 1.10 Modification Restrictions
**No changes after dispatch.** Once a document is sent for signing, additional signers cannot be added and the validity period cannot be changed.
